# Handoff: Cursor Tab 补全失效 → 汉化补丁根因

**Date:** 2026-05-26  
**Status:** ✅ 用户已确认通过 `cursor-i18n-tool --action=restore` 恢复英文后 Tab 补全正常  
**Next session focus:** 在另一窗口继续处理（例如：修复汉化工具避免误伤键位、清理调试期配置、或文档化）

---

## 问题摘要

用户在 Cursor 编辑器内按 **Tab** 无法接受 Cursor Tab / IntelliSense 补全，且焦点会跳到其他 UI 控件。  
`--classic` 启动、调整 `keybindings.json`、关闭 Tab Focus Mode 等均**未解决**。

**最终根因：** [cursor-i18n-tool](https://github.com/Wuyf5275/cursor-i18n-tool) 汉化补丁修改了 `workbench.desktop.main.js` 中的键盘扫描表，将物理键名 `"Tab"` 替换为 `"Tab 补全"`，导致键位服务无法分发 Tab（`Keyboard event cannot be dispatched`）。

---

## 关键证据（本机已验证）

路径：`/Applications/Cursor.app/Contents/Resources/app/out/vs/workbench/`

| 文件 | Tab 键盘表项 |
|------|----------------|
| `workbench.desktop.main.js.backup`（汉化前） | `[1,49,"Tab",2,"Tab",9,"VK_TAB"]` |
| `workbench.desktop.main.js`（汉化后） | `[1,49,"Tab 补全",2,"Tab 补全",9,"VK_TAB"]` |

- 存在 `.backup` 文件 → 确认曾运行过汉化工具  
- 汉化后 `Tab 补全` 在 workbench JS 中出现 **30 次**  
- 用户 renderer 日志典型模式：

```
[KeybindingService]: \ Keyboard event cannot be dispatched
Converted keydown event - keyCode: 2 ('Tab 补全')
\ Keyboard event cannot be dispatched in keydown phase.
```

对比：普通字母键有 `| Resolving [KeyL]`，Tab 则**无 Resolving**，与 Shift/Meta 修饰键同类失败。

汉化工具本地路径：`~/Projects/cursor-i18n-tool`

---

## 已排除 / 尝试过但未奏效的方向

1. 删除 `keybindings.json` 中对 `-editor.action.acceptCursorTabSuggestion` 的解绑（用户曾把 Tab 接受改到 Cmd+.）
2. `editor.tabFocusMode` / `editor.tabMovesFocus` / `accessibility.tabMovesFocus` 均为 `false`
3. Glass 模式：`--classic` 启动后问题仍在
4. 复杂 Tab 键位规则（auxiliaryBar、composerFocused 等）— 因根因是 chord 无法生成，键位规则无效
5. `cursor.general.disableHttp2` 从 `true` 改为 `false`（论坛 workaround，非本例根因）

---

## 已确认的有效修复

```bash
# 1. 完全退出 Cursor (Cmd+Q)
cd ~/Projects/cursor-i18n-tool
node index.js --action=restore
# 2. 重启 Cursor，在编辑器内测试 Tab
```

用户反馈：**还原英文后问题已解决。**

## 调试期改动的用户配置（可选清理）

若 Tab 已正常，下一窗口可考虑是否回滚这些「为排查而加」的项：

### `~/Library/Application Support/Cursor/User/settings.json`

- `cursor.general.disableHttp2`: `true` → 已改为 `false`
- 新增：`emmet.triggerExpansionOnTab: false`
- 新增：`editor.accessibilitySupport: "off"`
- 已有：`editor.tabFocusMode: false` 等（可保留）

### `~/Library/Application Support/Cursor/User/keybindings.json`

当前仍含 Tab 相关自定义（非根因，但可简化）：

- `cmd+.` → `acceptCursorTabSuggestion`（曾解绑 quickFix）
- `tab` → `acceptCursorTabSuggestion` / `acceptSelectedSuggestion` / `inlineSuggest.commit`
- `shift+tab` → `tab`（有补全时缩进用）
- `-editor.emmet.action.expandAbbreviation`

**建议：** 在英文恢复且 Tab 正常后，可尝试**清空 Tab 相关自定义**，仅保留 markdown 快捷键，恢复 Cursor 默认 Tab 行为。

## 推荐后续工作（供下一窗口）

### A. 修复 cursor-i18n-tool（若仍要汉化）

1. 在 `src/dict.js` / `src/i18n-core.js` 中**禁止**替换键盘扫描表、KeyCode 枚举、`VK_*` 相关字符串中的 `"Tab"`。
2. 将 `"Tab 补全"` 等设置文案限制在 **UI 可见字符串**（`children:`、`title:`、`markdownDescription` 等），沿用工具自带的 L4 risky 规则。
3. 汉化后做回归：开启 Keyboard Shortcuts Troubleshooting，按 Tab 应看到 `Resolving Tab` 且 `matched editor.action.acceptCursorTabSuggestion` 或 `tab`，**不应**出现 `cannot be dispatched`。
4. 向 upstream 提 issue/PR：https://github.com/Wuyf5275/cursor-i18n-tool

### B. 用户侧长期方案

- **推荐：** 仅用官方 `vscode-language-pack-zh-hans` 汉化 UI，**不要** patch `workbench.desktop.main.js`。
- 每次 Cursor 更新后若再运行汉化工具，需重新验证 Tab 键。

### C. 配置清理

- 评估是否恢复 `cursor.general.disableHttp2: true`（若用户原本为网络问题设置）
- 精简 `keybindings.json`
